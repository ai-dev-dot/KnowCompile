# AI 笔记 — 设计规格

## 项目定位

| 维度 | 决策 |
|------|------|
| 项目名 | AI 笔记 |
| 形态 | Electron 桌面应用（Windows / macOS / Linux） |
| 定位 | 中文优先、极简、纯个人使用的 LLM Wiki |
| 许可 | MIT |

AI 笔记是基于 Karpathy LLM Wiki 范式的个人知识管理工具：将原始资料"编译"成结构化 Wiki 页面，实现知识的持续积累，而非每次重新检索。

与现有项目（nashsu/llm_wiki 等）的差异化：更轻量、更极简、专注中文用户体验。

## 技术栈

| 层 | 选择 |
|----|------|
| 桌面框架 | Electron |
| 渲染进程 | React + Tailwind CSS，Vite 打包 |
| 主进程 | Node.js，直接操作文件系统 |
| 数据存储 | 纯 Markdown 文件，无数据库 |
| LLM 接口 | @anthropic-ai/sdk / openai 等 npm 包 |

选型理由：Electron 的 JS 生态与 LLM API 生态完全匹配，可直接使用 Anthropic、OpenAI 等官方 SDK。

## 数据架构：纯文件驱动

遵循 Karpathy 三层规范，Wiki 即 Markdown 文件：

```
用户的知识库/
├── raw/                ← 原始资料（LLM 只读）
├── wiki/               ← LLM 编译的知识页面（扁平结构）
│   ├── 量子计算.md
│   ├── 量子比特.md
│   └── 机器学习.md
├── schema/             ← AI 行为规则（人类可编辑）
│   ├── system.md       # 系统级指令
│   ├── compile-rules.md # 编译规则
│   ├── style-guide.md  # 文风与中文规范
│   └── links-rules.md  # 交叉链接规则
└── .ai-notes/          ← 应用元数据（搜索索引等，不提交到 wiki）
```

Wiki 页面采用**扁平 + 超链接**的维基百科式组织，页面之间通过 `[[双向链接]]` 表达关系，不使用文件夹层级。

## 应用架构

```
┌─────────────────────────────────┐
│         Electron Main           │
│  ┌───────────────────────────┐  │
│  │  FileSystem Manager       │  │
│  │  (raw/ wiki/ schema/ 读 写) │  │
│  ├───────────────────────────┤  │
│  │  LLM Service              │  │
│  │  (API 调用、流式响应)      │  │
│  ├───────────────────────────┤  │
│  │  Search Indexer           │  │
│  │  (内存索引，中文分词)      │  │
│  └───────────────────────────┘  │
│              ↕ IPC               │
│  ┌───────────────────────────┐  │
│  │    Electron Renderer       │  │
│  │    (React App)             │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

主进程通过 IPC 暴露 API 给渲染进程，渲染进程不直接访问文件系统。

## UI 设计

### 整体布局：三栏结构

- **左侧图标栏**（60px）：5 个核心视图切换 + 设置
- **页面列表**（220px，可折叠）：Wiki 页面 / 搜索结果 / raw 文件列表
- **内容区**（弹性）：Markdown 渲染 / 拖放摄入 / 问答对话 / 图谱画布
- **右侧面板**（200px，可收起）：反向链接、相关概念、页面元信息

### 五个核心视图

| 图标 | 视图 | 功能 |
|------|------|------|
| 📖 | Wiki 浏览 | 阅读编译后的知识页面，查看链接和反向链接 |
| 📥 | 资料摄入 | 拖放/粘贴文件，预览 raw/ 目录，触发编译 |
| 💬 | AI 问答 | 基于 Wiki 的对话式问答，答案附来源引用 |
| 🔗 | 知识图谱 | 页面关联网络可视化 |
| ⚙ | 设置 | API Key 配置、Schema 编辑、导出/备份 |

### 设计风格

- 默认暗色主题，适合长时间知识工作
- 中文排版优化：字体、行距、标点适配
- 关键概念高亮，矛盾用暖色标注
- 整体风格极简，去除不必要的装饰

## LLM 集成

### 支持的 API

- OpenAI（GPT-4o 等）
- Anthropic（Claude 系列）
- 兼容 OpenAI 接口的模型（DeepSeek、Qwen、MiniMax 等）
- API Key 由用户配置，存储在本地

### LLM 三种角色

| 角色 | 触发时机 | 输入 → 输出 |
|------|----------|-------------|
| 编译器 | 摄入新资料 | raw 资料 + 已有 wiki → 新/更新的 wiki 页面（含链接和矛盾标注） |
| 问答者 | 用户提问 | 问题 + 相关 wiki 页面 → 带引用的中文回答 |
| 维护者 | 手动/定期 | 全部 wiki → 链接建议、内容优化、矛盾检测 |

### 编译流程

1. 用户拖入资料 → App 存入 `raw/`
2. 用户点击"编译" → LLM 读取资料 + 现有 wiki 页面
3. LLM 提取关键概念 → 生成/更新 `wiki/*.md`
4. 自动标注来源、建立 `[[双向链接]]`
5. 检测与已有内容的矛盾 → 标注 `⚠ 矛盾提示`

## MVP 功能清单

| 功能 | 描述 |
|------|------|
| 资料摄入 | 拖放/粘贴文件（PDF、网页、Markdown、纯文本），自动存入 raw/ |
| LLM 编译 | 读取 raw/ 资料，自动生成结构化 Wiki 页面，交叉链接、矛盾标注 |
| Wiki 浏览 | 美观的 Markdown 渲染，页面导航、反向链接查看 |
| AI 问答 | 基于已编译 Wiki 的对话式问答，答案引用来源页面 |
| 全文搜索 | 搜索 Wiki 内容和原始资料，支持中文分词 |
| 知识图谱 | 页面关联网络可视化（类似 Obsidian Graph View） |
| 导出备份 | 导出为静态 HTML / PDF / Markdown，数据备份与恢复 |

MVP 不包含：本地模型（Ollama）支持（后续版本考虑）。

## Schema 规则系统

四个 Markdown 文件构成 AI 的行为规范，App 首次启动自动生成默认内容：

- **system.md** — 系统级指令（角色、语气、客观性原则）
- **compile-rules.md** — 编译触发条件、页面拆分粒度、矛盾检测
- **style-guide.md** — 页面格式模板、中文术语规范、引用格式
- **links-rules.md** — 交叉链接创建标准、页面分类规则

Schema 文件人类可编辑，每次 LLM 操作时都会读取。

## 不包含（YAGNI）

- 多用户 / 团队协作
- 本地模型（Ollama）集成（MVP 后评估）
- 网页剪藏（MVP 后评估）
- 插件系统
- 云同步
